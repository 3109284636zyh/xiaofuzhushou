# 产品展示库与小程序产品详情页设计

## 背景

当前项目已有后台“热卖产品库”、后端 `products` 表和小程序首页热卖产品模块。现有产品信息主要包含名称、类型、价格区间、开发周期、推荐描述、热卖状态、排序和状态，适合首页展示和报价辅助，但还不能满足“随时调取给客户”的销售场景。

本设计目标是把现有产品库升级为可运营的产品展示库：后台可自行编辑图文视频外链、卖点、适用场景、交付内容、常见问题和客户话术；小程序新增产品列表和产品详情页，方便把产品内容分享给客户，也方便复制话术用于平台内沟通。

## 已确认决策

- 展示位置：后台和小程序都要。
- 素材方式：先采用外链 URL，不做本地上传。
- 内容结构：图文视频 + 可复制客户话术。
- 调取方式：小程序分享 + 复制话术。
- 实现策略：扩展现有产品库，不新增独立产品素材库。

## 目标

- 后台“热卖产品库”升级为“产品展示库”，保留现有产品 CRUD 能力。
- 后台支持编辑产品封面图 URL、视频 URL、详情图 URL 列表、核心卖点、适用场景、交付内容、常见问题和客户话术。
- 小程序新增产品列表页，展示所有启用产品。
- 小程序新增产品详情页，展示图文视频、价格周期、卖点、场景、交付内容、详情图和常见问题。
- 首页热卖产品卡片可点击进入产品详情。
- 小程序详情页支持复制客户话术、复制产品摘要和转发给客户。
- 旧产品数据兼容：新增字段为空时页面不崩溃，并隐藏对应展示模块。

## 非目标

- 不做后台本地上传图片或视频。
- 不接入第三方对象存储。
- 不引入复杂富文本编辑器。
- 不做截图海报生成。
- 不做产品访问统计。
- 不做客户登录、支付、下单流程。
- 不重构报价系统核心逻辑。
- 不新增第三方 UI 组件库。

## 方案对比

### 方案 A：扩展现有热卖产品为产品展示库（采用）

在现有 `products` 表和后台产品管理基础上增加展示字段，小程序新增列表和详情页。

优点：复用现有产品数据、接口和首页热卖模块；开发量可控；不会出现两套重复产品数据。  
缺点：产品表字段会增加，后台表单需要分区整理。

### 方案 B：新增独立产品素材库

新增 `product_showcases` 之类的表和后台页面，产品展示素材与现有热卖产品分离。

优点：展示素材和报价产品解耦。  
缺点：会出现“产品”和“展示素材”两套内容，后续维护容易重复，不适合当前 MVP 阶段。

### 方案 C：只做后台素材库

只在后台维护图文视频和话术，不新增小程序产品详情。

优点：实现最快。  
缺点：客户无法直接通过小程序查看产品详情，不符合“后台 + 小程序都要”的需求。

## 数据模型

继续使用现有 `products` 表，新增字段：

- `cover_url`：封面图外链，字符串，可为空。
- `video_url`：视频外链，字符串，可为空。
- `gallery_urls`：详情图外链列表，JSON 字符串，默认为空数组。
- `highlights`：核心卖点列表，JSON 字符串，默认为空数组。
- `scenarios`：适用场景列表，JSON 字符串，默认为空数组。
- `deliverables`：交付内容列表，JSON 字符串，默认为空数组。
- `faqs`：常见问题列表，JSON 字符串，结构为 `{ question, answer }[]`，默认为空数组。
- `share_script`：客户话术，长文本，可为空。
- `detail_content`：补充详情说明，长文本，可为空。

兼容策略：

- 数据库新增字段全部允许为空或提供默认值，旧产品不需要立即补齐内容。
- 后端映射时把 JSON 字段统一解析成数组；解析失败时返回空数组。
- 后台保存时把多行文本列表转成数组，再由接口持久化为 JSON 字符串。
- 未填写 `share_script` 时，后端根据产品名称、描述、价格区间和周期生成默认话术。

## 后端接口

### 后台接口

复用现有后台产品接口，并扩展请求和响应字段：

- `GET /api/admin/products`
- `POST /api/admin/products`
- `PUT /api/admin/products/:id`
- `DELETE /api/admin/products/:id`

后台接口返回完整字段，包括图文视频、列表字段、FAQ、详情说明和客户话术。

### 小程序公开接口

新增或扩展公开接口：

- `GET /api/products`：产品列表，只返回启用产品，按 `sort_order DESC, id DESC` 排序。
- `GET /api/products/:id`：产品详情，只允许查看启用产品。
- `GET /api/products/hot`：保留现有首页热卖接口，补充 `coverUrl` 等轻量展示字段。

公开接口不返回禁用产品。若产品不存在或已禁用，详情接口返回清晰错误，由小程序展示“产品暂不可查看”。

## 后台页面设计

现有后台 [Products.vue](../../../admin/src/views/Products.vue) 从“热卖产品库”升级为“产品展示库”。列表保留现有列，并可增加封面、热卖、状态等展示信息。新增和编辑仍使用弹窗或抽屉，表单分区展示。

### 基础信息

- 产品名称
- 产品类型
- 价格区间
- 开发周期
- 是否热卖
- 排序
- 状态

### 展示素材

- 封面图 URL
- 视频 URL
- 详情图 URL 列表

详情图列表在后台用“一行一个图片链接”的方式编辑，保存时转成数组。

### 产品内容

- 推荐描述
- 详情说明
- 核心卖点列表
- 适用场景列表
- 交付内容列表

核心卖点、适用场景和交付内容先用多行文本编辑，一行一条。这样比引入复杂富文本编辑器更稳定，也更适合小程序结构化展示。

### 客户调取

- 客户话术
- 不填写时使用后端默认话术

后台可提供“复制话术”按钮，方便在后台直接调取内容。

### 常见问题

- 支持多条问题和回答。
- 后台用简单动态表单维护，不引入第三方编辑器。

## 小程序页面设计

### 产品列表页

新增页面：

- [products.js](../../../miniprogram/pages/products/products.js)
- [products.wxml](../../../miniprogram/pages/products/products.wxml)
- [products.wxss](../../../miniprogram/pages/products/products.wxss)
- [products.json](../../../miniprogram/pages/products/products.json)

入口：

- 首页快捷入口“热卖产品”跳转到产品列表页。
- 后续搜索结果也可以跳转产品详情页。

页面内容：

- 顶部说明：产品展示库 / 常用建站服务。
- 产品卡片：封面、名称、描述、价格区间、开发周期、热卖标签。
- 空态：暂无产品展示。
- 点击产品卡片进入产品详情页。

### 产品详情页

新增页面：

- [product-detail.js](../../../miniprogram/pages/product-detail/product-detail.js)
- [product-detail.wxml](../../../miniprogram/pages/product-detail/product-detail.wxml)
- [product-detail.wxss](../../../miniprogram/pages/product-detail/product-detail.wxss)
- [product-detail.json](../../../miniprogram/pages/product-detail/product-detail.json)

页面结构：

1. 顶部封面：有封面图时展示图片，没有封面时展示蓝紫渐变默认卡片。
2. 基础信息：产品名称、推荐描述、价格区间、开发周期。
3. 视频展示：有 `videoUrl` 时显示视频模块，没有则隐藏。
4. 核心卖点：用卡片或标签列表展示。
5. 适用场景：帮助客户判断是否适合自己的业务。
6. 交付内容：说明客户最终能拿到什么。
7. 详情图：展示后台维护的图片 URL 列表。
8. 常见问题：普通问答列表，先不做复杂折叠组件。
9. 底部操作栏：复制话术、复制摘要、转发给客户。

页面视觉沿用当前小程序蓝紫渐变、玻璃卡片、圆角卡片和本地图标体系。

## 复制与分享

### 复制话术

复制后台维护的 `shareScript`。如果为空，复制后端生成的默认话术。成功后提示“已复制，可直接发给客户”。

### 复制摘要

小程序按当前详情数据拼接摘要：

- 产品名称
- 价格区间
- 交付周期
- 3 条以内核心卖点
- 简短推荐描述

用于快速回复客户，不依赖后台单独维护。

### 转发给客户

详情页实现 `onShareAppMessage`：

- 分享标题使用产品名称。
- 分享路径包含产品 ID，例如 `/pages/product-detail/product-detail?id=1`。
- 分享图优先使用封面图；无封面时使用默认分享信息。

## 错误处理

### 后台

- 产品列表加载失败：显示错误提示，保留页面空态。
- 产品保存失败：保留当前弹窗内容，提示错误信息。
- 删除产品：沿用现有二次确认。
- URL 字段：允许为空；实施时可做轻量提示，但不阻断保存。
- JSON 列表字段解析失败：后端返回空数组，避免后台和小程序崩溃。

### 小程序

- 产品列表加载失败：显示空态和重新加载入口。
- 产品详情加载失败：显示错误卡片和返回按钮。
- 产品不存在或禁用：显示“产品暂不可查看”。
- 封面图加载失败：退回默认渐变卡片。
- 视频 URL 为空：隐藏视频模块。
- 复制成功：提示“已复制，可直接发给客户”。
- 复制失败：提示“复制失败，请手动选择内容”。

## 测试与验收

### 后端

- 产品字段映射覆盖新增字段。
- JSON 字段能正确序列化和解析。
- JSON 字段异常时返回空数组。
- `GET /api/products` 只返回启用产品。
- `GET /api/products/:id` 能返回启用产品详情。
- 禁用产品或不存在产品不可公开查看。
- 未填写客户话术时能生成默认话术。

### 后台

- 能创建含封面、视频、详情图、卖点、场景、交付内容、FAQ、话术的产品。
- 能编辑并保存新增字段，刷新后仍可见。
- 旧产品新增字段为空时仍能正常编辑、保存和删除。
- 后台复制话术可用。

### 小程序

- 产品列表能加载启用产品。
- 首页热卖产品卡片可进入对应产品详情。
- 快捷入口“热卖产品”进入产品列表页。
- 产品详情能展示封面、视频、详情图、卖点、场景、交付内容和 FAQ。
- 未填写视频、图片或 FAQ 时隐藏对应模块，不出现空白异常。
- 复制话术和复制摘要能写入剪贴板。
- 分享路径包含产品 ID。

## 验收标准

- 后台可以完整维护一个产品的图文视频外链、结构化内容、FAQ 和客户话术。
- 小程序可以从首页进入产品列表和详情。
- 产品详情页可以直接复制话术、复制摘要，并通过小程序转发给客户。
- 没有图片、视频或 FAQ 的旧产品仍能正常展示，不出现空白模块或报错。
- 不依赖本地上传和第三方素材服务即可上线使用。
