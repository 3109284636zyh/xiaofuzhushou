# Admin 账号密码登录与 API 配置弱化设计

日期：2026-06-21  
目标：完善 `admin/` 网页后台登录体验，使后台仅使用固定账号密码登录，并弱化 DeepSeek API 配置入口带来的“必须先填 API 才能用后台”的感知。

## 1. 已确认决策

- 后台登录改为账号 + 密码。
- 固定账号：`admin`。
- 固定密码：`aw3109284636`。
- 保留后端托管 DeepSeek API Key 的能力，但不再把 API 配置作为后台主要入口强调。
- 不引入注册、用户管理、多角色权限、找回密码或验证码。

## 2. 推荐方案

采用“账号密码登录 + 保留但弱化 AI 配置”的方案。

登录页展示账号和密码输入框，默认提示账号为 `admin`。后端登录接口同时校验 `username` 和 `password`，校验成功后继续返回原有 JWT。后台菜单移除或弱化侧边栏中的“AI 配置”主入口，避免用户进入后台后被引导去填写 API；需要时仍可通过后端环境变量或保留页面维护 DeepSeek 配置。

## 3. 后端设计

### 3.1 登录凭据

在后端常量中维护固定账号和密码：

- `ADMIN_USERNAME = 'admin'`
- `ADMIN_PASSWORD = 'aw3109284636'`

这保持当前项目“轻量固定后台账号”的 MVP 模式，不新增数据库账号表和密码加密流程。

### 3.2 登录接口

更新 `POST /api/auth/login` 请求体：

```json
{
  "username": "admin",
  "password": "aw3109284636"
}
```

后端同时校验账号和密码：

- 账号或密码缺失：返回参数错误或统一登录失败提示。
- 账号不是 `admin` 或密码不匹配：返回“账号或密码错误”。
- 校验成功：返回现有 JWT、过期时间和管理员名称。

JWT 结构继续保持轻量：

```json
{
  "role": "admin",
  "adminName": "AI小福管理员",
  "username": "admin"
}
```

## 4. 前端后台设计

### 4.1 登录页

`admin/src/views/Login.vue` 增加账号输入框：

- 账号输入框默认值为 `admin`，placeholder 提示“请输入账号”。
- 密码输入框继续使用 password 类型。
- 登录按钮文案保持“进入中控台”。
- 顶部说明改为“使用后台账号密码登录，AI 能力由后端托管”。

登录时调用 `login(username, password)`，由 API 模块传给后端。

### 4.2 API 模块

`admin/src/api/modules.js` 中的登录函数从 `login(password)` 改为 `login(username, password)`，请求体传 `{ username, password }`。

### 4.3 弱化 API 配置入口

侧边栏不再突出显示“AI 配置”。推荐做法：

- 从 `admin/src/App.vue` 主菜单移除“AI 配置”。
- 顶部状态文案保留“DeepSeek 由后端托管”，让用户知道不需要在登录时填写 API。
- `AiConfig.vue` 文件和路由可暂时保留，避免删除已有能力；只是从主导航中弱化入口。

## 5. 数据流

1. 管理员打开后台。
2. 路由守卫发现没有 token，跳转 `/login`。
3. 管理员输入账号 `admin` 和密码 `aw3109284636`。
4. 前端调用 `POST /api/auth/login`。
5. 后端校验固定账号密码。
6. 登录成功后返回 JWT。
7. 前端保存 token，跳转仪表盘。
8. 后续后台接口继续通过 `Authorization: Bearer <token>` 调用。

## 6. 错误处理

- 空账号：前端提示“请输入后台账号”。
- 空密码：前端提示“请输入后台密码”。
- 后端校验失败：返回“账号或密码错误”。
- JWT 过期：沿用现有 `40101` 处理，清理 token 并跳转登录页。

## 7. 测试与验证

需要验证：

1. 后端测试或接口测试：
   - `admin` + `aw3109284636` 登录成功。
   - 错误账号登录失败。
   - 错误密码登录失败。
2. 前端构建：`npm run admin:build` 通过。
3. 后端测试：`npm run backend:test` 通过。
4. 手动体验：登录页不再只出现密码，也不再强调必须填写 API。

## 8. 非目标

本次不做：

- 数据库管理员账号表。
- 多管理员、多角色权限。
- 密码哈希或密码修改页面。
- 验证码、短信、扫码登录。
- 删除 DeepSeek 配置接口。
