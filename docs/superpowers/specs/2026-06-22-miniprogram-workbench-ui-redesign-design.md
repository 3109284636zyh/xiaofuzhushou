# 小程序工作台风格统一改造设计

日期：2026-06-22  
范围：微信小程序全局页面视觉与布局统一，覆盖首页、我的页、建站助手、工具箱、搜索、报价、方案、AI 聊天、文章详情等主要页面。  
目标：参考用户提供的“名片君”首页和我的页截图，把 AI 小福小程序统一升级为更成熟的工作台式 UI；保留“AI 建站接单助手”产品定位和现有业务接口，不改成名片管理产品。

## 1. 背景

当前小程序已经具备蓝紫渐变、圆角卡片、本地图标和自定义导航安全区处理。近期已完成首页首屏优化和全局 UI 图标补齐，但各页面仍存在以下问题：

1. 页面结构不够统一：首页、我的页、工具箱、搜索页和二级页的卡片形态、入口密度、分组方式不一致。
2. 首页不像一个完整工作台：已有 hero、搜索、快捷入口和内容列表，但参考图中的“大工作台 hero + 入口宫格 + 当前主内容卡片”层级更清晰。
3. 我的页不够产品化：当前是配置状态和菜单列表，参考图中的资料卡、统计条、分组菜单更像正式用户中心。
4. 其他页面视觉承载偏散：建站助手、工具箱、报价、方案、AI 聊天、文章详情已经补图标，但缺少统一的工作台外壳。

本次设计采用用户确认的方案 A：**全局“名片君式工作台外壳”，业务仍保持 AI 建站接单助手**。

## 2. 设计目标

1. 全小程序向参考图的工作台风格靠拢：大圆角白卡、浅蓝灰背景、蓝紫重点卡片、分组列表、宫格入口。
2. 首页接近参考图 1 的结构感：顶部大 hero、四个核心入口、当前主推服务/数据/最近内容。
3. 我的页接近参考图 2 的结构感：资料卡、四项统计条、分组菜单。
4. 其他页面统一视觉语言，不强行改变业务流程。
5. 保留现有接口、数据字段、跳转语义和表单校验。
6. 页面继续可上下滚动，内容不被 tabBar、状态栏或微信胶囊按钮遮挡。
7. 不引入第三方 UI 库，不新增自定义 tabBar，不新增登录、会员、访客线索等业务。

## 3. 非目标

- 不把产品定位改成名片管理工具。
- 不新增真实名片、访客线索、会员权益、二维码客户沉淀等数据模型。
- 不修改后端 API、数据库、管理后台。
- 不引入 Vant、TDesign、iconfont 或其他第三方 UI 组件库。
- 不重新启用 Skyline 渲染器；当前页面以普通 view 页面滚动为基础。
- 不做复杂动画或视觉特效，优先保证页面稳定、可读、可滚动。

## 4. 已确认方案

采用方案 A：**统一工作台视觉，业务不改**。

- 首页：按参考图 1 改为“大渐变工作台 hero + 核心能力卡 + 四宫格入口 + 内容分区”。
- 我的页：按参考图 2 改为“用户资料卡 + 统计条 + 分组菜单”。
- 建站助手、工具箱、搜索：保留业务结构，统一顶部 hero、卡片、入口、列表行。
- 报价、方案、AI 聊天、文章详情：统一表单卡、结果卡、阅读卡和底部安全距离。
- 全局：通过 `app.wxss` 增加工作台样式工具类，各页面 WXML 接入，避免新增自定义组件导致改动过大。

## 5. 全局视觉系统

### 5.1 背景

页面背景从当前单一渐变感，调整为参考图类似的浅蓝灰工作台底色：

- 主背景建议：`#F4F7FC` / `#EEF4FF`。
- 顶部可保留轻微蓝色氛围渐变。
- 页面根容器统一包含底部 tabBar 安全留白。
- 保持 `navigationStyle: custom`，继续使用顶部安全区 spacer 避开状态栏和胶囊按钮。

### 5.2 卡片

卡片统一更接近参考图的大圆角白卡：

- 普通卡片圆角：约 `28rpx` - `36rpx`。
- 重点 hero 圆角：约 `40rpx`。
- 阴影降低透明度，使用柔和浮层，不做厚重玻璃拟态。
- 边框只在需要区分层级时使用极浅色，如 `rgba(22,119,255,.08)`。

### 5.3 色彩

- 主色继续使用 `#1677FF`。
- 辅助紫色继续使用 `#6C63FF`。
- 文字主色：接近 `#111827` / `#172033`。
- 次级文字：接近 `#7A8497` / `#8A94A6`。
- 入口图标底：浅蓝、浅紫、浅绿、浅黄、浅粉等弱色块，避免全页只有蓝紫。

### 5.4 字体层级

- hero 主标题：约 `42rpx` - `48rpx`，粗体。
- 页面/分组主标题：约 `34rpx` - `40rpx`，粗体。
- 卡片标题：约 `28rpx` - `32rpx`，粗体。
- 描述文字：约 `23rpx` - `26rpx`，常规或中等字重。
- 标签/徽标：约 `22rpx` - `24rpx`。

## 6. 全局样式抽象

不新增小程序自定义组件，先在 `miniprogram/app.wxss` 中新增或强化一组复用类。

建议新增：

- `.workbench-page`：统一浅蓝灰背景、最小高度、底部安全区和水平间距基调。
- `.workbench-hero`：大渐变 hero 卡片，支持装饰圆形、徽标、标题、副标题。
- `.workbench-hero-orb`：hero 装饰圆形，必须不遮挡主要文字。
- `.workbench-badge`：状态标签，如“后端安全托管”“风控已启用”。
- `.workbench-grid`：入口宫格容器，支持 2 列、3 列或 4 列。
- `.workbench-entry-card`：入口卡片，白底、居中图标、标题、说明。
- `.workbench-section`：分组容器。
- `.workbench-section-title`：统一分组标题。
- `.workbench-list-card`：整组列表白卡。
- `.workbench-list-row`：菜单行、搜索结果、记录行。
- `.workbench-icon-tile`：左侧浅色图标块。
- `.workbench-stat-strip`：我的页统计条。
- `.workbench-stat-item`：统计条单项。
- `.workbench-form-card`：报价、方案、聊天等表单卡。
- `.workbench-result-card`：结果展示卡。

这些类和已有 `.card`、`.pill`、`.btn-primary`、`.ui-icon`、`.title-with-icon`、`.row-with-icon` 可以并存。实施时优先复用已有类，避免全量重写。

## 7. 页面改造设计

### 7.1 首页

首页最接近参考图 1，但保留 AI 建站助手定位。

结构建议：

1. 顶部安全 spacer。
2. 大 hero 工作台卡：
   - 标签：`AI小福工作台` 或 `平台内接单助手`。
   - 主标题：突出“今天帮你快速生成报价和方案”。
   - 副标题：突出“DeepSeek 后端托管 · 平台沟通风控已启用”。
   - 状态徽标：如“已启用”“安全托管”。
   - 内部两个核心能力卡：`AI 回复`、`智能报价`，用于模拟参考图中的两个大入口。
3. 四宫格入口：
   - AI智能回复 → `/pages/ai-chat/ai-chat`。
   - 智能报价 → `/pages/quote/quote`。
   - 建站百科 → 建站助手 tab。
   - 热卖产品 → 首页当前产品分区。
4. 今日数据：改为更紧凑的工作台统计条或数据卡。
5. 内容区：
   - `当前主推`：优先展示热卖产品第一项或热卖产品列表。
   - `最新教程`：展示教程列表。
   - `最近使用`：展示最近记录或统一空态。

行为保持不变：保留 `goSearch`、`handleQuick`、`loadHome` 等已有逻辑。

### 7.2 我的页

我的页参考图 2，做成用户资料工作台。

结构建议：

1. 顶部安全 spacer。
2. 用户资料白卡：
   - 头像图标或“福”字头像。
   - 标题：`AI小福工作台` 或 `AI小福用户`。
   - 副标题：`无注册 · 无付费 · 后端安全托管 API`。
   - 右上状态标签：`免费版` 或 `已启用`，使用固定文案表达，不新增会员体系。
3. 四项统计条：
   - `API`：后端安全托管。
   - `风控`：已启用。
   - `收藏`：点击后加载收藏。
   - `历史`：点击后加载历史。
   如果某些统计没有真实数量，使用状态文字或入口文字，不伪造动态数字。
4. 分组菜单：
   - `名片管理`改为适合本项目的`工作台管理`或`常用入口`。
   - 保留现有菜单项：收藏内容、历史记录、AI 配置、API 管理、更新日志、联系客服。
   - 可按 `常用入口`、`服务与数据`、`关于与配置` 分组。
5. 记录列表：点击收藏或历史后，仍展示现有 records，样式改为统一工作台列表行。

行为保持不变：保留 `openMenu`、`openFavorites`、`openHistory`。

### 7.3 建站助手页

建站助手页不改业务流，只统一视觉外壳。

- 顶部 hero：改为轻量工作台 hero，突出“AI 建站导师”。
- 输入区域：保留 textarea 和生成按钮，放入 `.workbench-form-card`。
- AI 回复：统一 `.workbench-result-card`，复制按钮更清晰。
- 知识分类：分类 chip 保留横向滚动，样式与工作台入口一致。
- 文章列表：统一列表行，左侧 icon tile，中间标题摘要，底部标签。

### 7.4 工具箱页

工具箱保留九宫格和智能接单行为，视觉参考首页入口卡。

- 顶部 hero：突出“报价、方案、分析和检测入口一次放齐”。
- 工具入口：继续 3 列或按屏宽调整为 3 列，卡片更接近参考图入口卡。
- 智能接单：使用分组列表卡展示“客户需求分析”“销售话术推荐”。
- 结果面板：统一结果卡样式，行间分割更轻。

### 7.5 搜索页

搜索页保留关键词搜索和结果跳转。

- 顶部 hero 或搜索头卡：突出“产品、知识、教程和历史记录一起搜”。
- 搜索输入区：白色圆角输入卡，按钮与输入不拥挤。
- 空态：统一浅色空态卡。
- 结果：统一列表行，左侧语义图标，中间类型、标题、摘要，右侧箭头。

### 7.6 报价页和方案页

报价、方案页是表单 + 结果结构。

- 顶部 hero：简短说明页面能力。
- 表单区域：统一 `.workbench-form-card`，输入框间距一致。
- 提交按钮：继续使用主按钮，可保留图标。
- 结果区域：统一 `.workbench-result-card`，价格、周期、功能列表层级更明显。
- 复制按钮：保持现有逻辑。

### 7.7 AI 聊天页

AI 聊天页保持单输入、生成回复、复制回复流程。

- 顶部 hero：突出“平台内安全回复”。
- 输入卡：更大留白，textarea 与图标对齐。
- 结果卡：回复内容可复制，兜底提示使用浅黄色提示条。

### 7.8 文章详情页

文章详情页做阅读体验优化。

- 顶部安全 spacer 保留。
- 标题区：白色阅读卡，显示标题、摘要、标签或分类信息。
- 正文：使用 `.copy-text` 或阅读正文样式，行高保持舒适。
- 收藏按钮：作为主操作或轻按钮，位置清晰。
- 如果文章为空，展示统一空态。

## 8. 数据流与接口

所有现有接口保持不变：

- 首页继续请求 `/app/home/summary`、`/products/hot`、`/tutorials/latest`、`/usage/recent`。
- 建站助手继续请求知识分类、文章和 AI 回复接口。
- 工具箱继续调用工具检测、客户分析、话术推荐接口。
- 搜索继续调用 `/search`。
- 报价继续调用 `/quote/generate`。
- 方案继续调用 `/plan/generate`。
- AI 聊天继续调用 `/ai/chat`。
- 我的页继续调用 `/favorites` 和 `/history`。
- 文章详情继续调用 `/knowledge/articles/:id` 和 `/favorites`。

参考图中存在但本项目没有的数据，如访客线索、名片数量、会员等级，不新增后端字段。需要表达时使用现有状态字段或固定产品文案。

## 9. 错误处理与空态

- 当前接口失败多为静默处理，本次不扩大错误处理范围，避免改变用户体验和业务语义。
- 空列表使用统一空态卡片，文案与当前页面语义匹配。
- 表单校验保持现状，如报价页需求为空提示、方案页行业和需求为空提示、AI 聊天输入为空提示。
- 页面应在接口返回空数组、null 或请求失败时保持布局稳定。

4. 首页、我的页、建站助手、工具箱、搜索、报价、方案、AI 聊天、文章详情引入工作台样式 class。

7. 原有跳转和按钮行为不丢失。

## 12. 风险与约束

1. 全局样式影响面较大：实施时应优先新增工作台类，不直接重写所有 `.card` 基础语义，降低回归风险。
2. 页面 WXML 当前部分为单行密集结构，改造时需要小心保持事件绑定和数据字段不变。
3. 参考图包含名片管理、访客线索、会员等业务，本项目没有这些数据，设计上必须避免误导用户以为有这些功能。
4. 微信小程序不同机型的状态栏和底部安全区差异较大，必须保留现有 top spacer 和底部 padding 机制。
5. 过多阴影、滤镜或复杂层叠可能影响小程序性能，视觉效果以稳定为先。

## 13. 实施边界

实施阶段应分批完成：

1. 先新增全局工作台样式和回归测试。
2. 再改首页和我的页，因为这两个页面最能体现参考图效果。
3. 然后统一建站助手、工具箱、搜索三个核心 tab。
4. 最后统一报价、方案、AI 聊天、文章详情等二级页面。

每批修改后运行静态测试，避免一次性大改导致难以定位问题。

## 14. 验收结论标准

本设计完成后的可接受结果是：

- 用户打开小程序时，第一眼感觉像一个成熟的 AI 工作台，而不是分散的功能页面集合。
- 首页和我的页明显能看出参考图的结构借鉴，但文案和功能仍属于 AI 建站接单助手。
- 全小程序视觉统一，主要页面的卡片、入口、列表、标题和图标风格一致。
- 页面滚动、安全区、tabBar 遮挡问题不复发。
