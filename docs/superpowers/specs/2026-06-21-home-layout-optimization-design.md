# 首页布局优化设计规格

日期：2026-06-21  
范围：微信小程序首页 `miniprogram/pages/home/*` 视觉与布局优化  
目标：解决首页内容顶到状态栏/刘海区域、首屏视觉拥挤、卡片层级松散的问题，让首页更像一个清爽的 AI 建站接单工作台。

## 1. 背景

当前首页使用 `navigationStyle: custom`，页面内容从顶部直接开始渲染。在 WebView 预览里，hero 卡片贴近状态栏和右上角胶囊按钮，首屏显得压迫。同时 hero 标题字号偏大、文案偏长，搜索框和快捷入口卡片都比较厚重，导致整体观感不够精致。

本次只优化小程序首页的布局和视觉节奏，不改变后端接口、不改变 tabBar 结构、不扩展业务功能。

## 2. 目标

1. 首页内容避开状态栏、刘海和右上角胶囊按钮区域。
2. 首屏 hero 更克制，信息层级更清楚。
3. 搜索框、快捷入口、数据区和列表区的间距统一。
4. 页面底部为 tabBar 和安全区留出空间。
5. 保留当前蓝紫 AI 助手风格，但减少大白块堆叠和过重阴影。

## 3. 非目标

- 不新增自定义导航栏组件。
- 不改全局路由、tabBar 页面数量或接口字段。
- 不引入第三方 UI 库。
- 不改管理后台和后端逻辑。
- 不做完整品牌视觉重设计。

## 4. 布局方案

### 4.1 顶部安全区

首页 `home.js` 在加载时读取系统信息和右上角胶囊按钮位置，计算顶部内容偏移量。页面根节点通过内联样式或数据绑定应用该偏移，确保 hero 卡片从安全区域下方开始。

推荐数据字段：

- `topSpacerHeight`：顶部安全区高度，包含状态栏与胶囊按钮下沿后的间距。

计算策略：

- 优先使用 `wx.getMenuButtonBoundingClientRect()` 获取胶囊按钮位置。
- 使用 `wx.getSystemInfoSync()` 或兼容 API 获取 `statusBarHeight`。
- 顶部内容起点建议为 `menuButton.bottom + 16rpx` 对应的安全距离。
- 如果 API 不可用，使用保守默认值，避免内容贴顶。

### 4.2 Hero 区

Hero 保留蓝紫渐变，但降低压迫感：

- 减少上下 padding。
- 主标题控制为两行以内。
- 副标题使用较小字号和较低透明度。
- 背景装饰圆形弱化，不遮挡文字。
- 卡片圆角和阴影保持柔和。

建议文案：

- 标签：`平台内接单助手`
- 标题：`今天帮你快速生成报价和方案`
- 副标题：`DeepSeek 后端托管 · 平台沟通风控已启用`

### 4.3 搜索框

搜索区域从普通白卡片调整为胶囊搜索入口：

- 左侧加搜索符号。
- 高度更紧凑。
- 背景使用半透明白色或纯白。
- 阴影弱化，边框更细。
- 点击行为保持跳转到搜索 tab。

### 4.4 快捷入口

四个快捷入口继续使用两列布局，但优化卡片密度：

- 统一卡片最小高度。
- 图标使用小型圆形底，减少单个字符裸露的粗糙感。
- 标题和描述字号分层。
- 卡片间距使用统一 token，减少松散感。

点击行为保持不变：

- AI 智能回复 → `pages/ai-chat/ai-chat`
- 智能报价 → `pages/quote/quote`
- 建站百科 → 建站助手 tab
- 热卖产品 → 首页

### 4.5 今日数据

今日数据从三个独立厚卡片调整为一张横向数据面板，内部三列：

- 数字突出显示。
- 标签弱化。
- 列之间使用细分割线或自然间距。

这样可以减少白卡片堆叠，让页面更像工作台仪表盘。

### 4.6 内容列表

热卖产品、最新教程、最近使用保持原有数据来源和渲染逻辑，但卡片更轻：

- 减少 padding 和阴影。
- 标题、摘要、价格层级更清楚。
- 列表卡片之间间距更小。
- 空状态保持，但视觉上与新版页面协调。

### 4.7 底部安全区

首页根容器增加底部 padding，避免最后内容贴近 tabBar 或被底部安全区遮挡。

## 5. 文件影响

主要修改：

- `miniprogram/pages/home/home.js`
  - 增加顶部安全区计算。
  - 保持现有数据加载和快捷跳转逻辑。
- `miniprogram/pages/home/home.wxml`
  - 增加顶部 spacer。
  - 调整 hero、搜索框、快捷卡片、数据面板的结构 class。
- `miniprogram/pages/home/home.wxss`
  - 重写首页局部样式，优化间距、字号、阴影、卡片密度。

可能微调：

- `miniprogram/app.wxss`
  - 如全局 `.container` padding 与新版首页冲突，可让首页使用自己的 `.home-page` spacing，不影响其他页面。

## 6. 兼容性和风险

- `wx.getMenuButtonBoundingClientRect()` 在微信小程序环境可用；若预览环境返回异常，需要 fallback。
- Skyline 渲染下应避免过度依赖复杂 CSS 特性，本次主要使用普通布局、渐变、圆角和阴影。
- 首页接口返回空数据时，布局仍应保持稳定。
- 顶部安全区只在首页处理，避免影响其他页面当前表现。

## 7. 验证标准

1. WebView/微信开发者工具预览中，hero 不再顶到状态栏、刘海或胶囊按钮。
2. 首屏标题、搜索、快捷入口都在视觉安全区域内。
3. 今日数据显示为紧凑横向面板。
4. 热卖产品、最新教程、最近使用能正常显示空态或列表。
5. 首页跳转行为保持不变。
6. 其他 tab 页面不因本次首页样式修改出现明显布局错乱。
