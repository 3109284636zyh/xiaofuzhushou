# AI小福智能建站助手 V3.0 全栈 MVP 设计规格

日期：2026-06-20  
来源：`AI小福V3.0优化开发方案.txt` 与已确认需求  
目标：从零制作一个可运行、可扩展、可部署的 AI 小福 V3.0 模块化全栈 MVP。

## 1. 已确认决策

- 项目从零搭建完整工程，不基于现有源码改造。
- 第一阶段交付“可运行全栈 MVP”。
- 开发路线采用“模块化全栈 MVP”：小程序、后台、后端、数据库同时搭建骨架并跑通核心业务闭环。
- AI 服务使用 DeepSeek API。
- 对外部署域名沿用 `wfr.ccvo.top`。
- 小程序不做用户注册、登录、付费。
- 后台登录密码固定为 `zyh123456`。
- DeepSeek API Key 存在后端环境变量或后台系统配置中，小程序端不暴露完整密钥。

## 2. MVP 范围

MVP 需要优先跑通以下核心闭环：

1. 首页工作台：展示快捷功能、统计数据、热卖产品、最新教程、最近使用记录。
2. 建站助手：支持知识分类、文章浏览、AI 客服回复生成。
3. 工具箱：实现智能报价、AI 方案生成、客户需求分析、销售话术推荐；其他检测类工具先做可扩展入口。
4. 搜索：统一搜索产品、知识库、教程、历史记录。
5. 我的：展示收藏、历史、AI 配置/API 管理入口、更新日志、联系客服入口。
6. 管理后台：固定密码登录后管理产品、知识库、教程、话术、订单、AI 配置和基础统计。
7. 后端 API：统一支撑小程序和后台，封装 DeepSeek 调用、风控过滤、统计记录、MySQL 数据访问。
8. 数据库：提供建表脚本和初始化数据，保证项目拉起后有可演示内容。

MVP 不强行接入 Whois、备案查询、网站测速、SEO 检测等第三方真实服务。这些功能在首版提供入口、统一接口结构、模拟结果或“即将接入”提示，避免外部服务阻塞第一阶段交付。

## 3. 项目结构

```text
WANGFURONG/
├─ miniprogram/        微信原生小程序，按 Skyline 风格设计页面
├─ admin/              Vue3 + Element Plus 管理后台
├─ backend/            Node.js + Express 后端 API
├─ database.sql        MySQL 建表与初始化数据
├─ docs/               设计文档、接口说明、部署说明
└─ README.md           本地运行与项目说明
```

## 4. 小程序端设计

### 4.1 底部导航

小程序包含 5 个底部 Tab：

1. 首页
2. 建站助手
3. 工具箱
4. 搜索
5. 我的

UI 风格遵循开发方案：现代 AI 助手风格，主色使用 `#1677FF`、`#409EFF`、`#6C63FF`，卡片式布局，20px 圆角，适度毛玻璃效果，并预留深浅色模式样式变量。

### 4.2 首页

首页定位为工作台和快速入口。

模块：

- 顶部欢迎区：展示“AI小福智能建站助手”和今日状态。
- 搜索框：跳转全局搜索页。
- 四宫格快捷功能：AI 智能回复、智能报价、建站百科、热卖产品。
- 数据统计：今日使用次数、回复生成数量、成交客户数量。
- 热门推荐：热卖产品、最新教程。
- 最近使用记录：展示最近 AI 回复、报价、方案生成记录。

主要接口：

- `GET /api/app/home/summary`
- `GET /api/products/hot`
- `GET /api/tutorials/latest`
- `GET /api/usage/recent`

### 4.3 建站助手

建站助手是知识库和 AI 问答中心。

分类：

- 建站基础
- 域名服务器
- WordPress
- SEO 优化
- AI 网站
- 微信小程序

功能：

- 分类浏览知识文章。
- 查看文章详情。
- 输入客户问题，调用 DeepSeek 生成平台内可用客服回复。
- 生成结果可复制、收藏、保存历史。

主要接口：

- `GET /api/knowledge/categories`
- `GET /api/knowledge/articles`
- `GET /api/knowledge/articles/:id`
- `POST /api/ai/chat`
- `POST /api/favorites`
- `POST /api/history`

### 4.4 工具箱

工具箱采用九宫格布局。

MVP 真实可用工具：

- 智能报价
- AI 方案生成
- 网站需求分析
- 销售话术推荐

MVP 预留扩展工具：

- 域名查询
- Whois 查询
- 网站测速
- SSL 检测
- 备案查询
- SEO 检测
- 网站分析

主要接口：

- `POST /api/quote/generate`
- `POST /api/plan/generate`
- `POST /api/customer/analyze`
- `POST /api/scripts/recommend`
- `POST /api/tools/:toolKey/check`

扩展工具首版返回模拟结果或“功能已预留，即将接入”，不返回前端错误。

### 4.5 搜索页

全局搜索支持：

- 产品
- 知识库
- 教程
- 客户问题历史
- 使用记录

接口：

- `GET /api/search?q=关键词&type=all`

统一结果格式：

```json
{
  "type": "product | article | tutorial | history",
  "title": "标题",
  "summary": "摘要",
  "targetId": 1
}
```

### 4.6 我的

我的页作为工具配置中心，不引入真实用户账号体系。

包含：

- 个人信息展示卡片。
- 收藏内容。
- 历史记录。
- AI 配置入口。
- API 管理入口。
- 更新日志。
- 联系客服。

小程序端只展示 API 配置状态，不展示完整 DeepSeek Key。

## 5. 管理后台设计

后台使用 Vue3 + Element Plus 构建单页面应用。

### 5.1 登录

- 固定密码：`zyh123456`。
- 登录成功后返回 JWT。

- 登录过期后自动跳回登录页。

### 5.2 仪表盘

展示：

- 今日报价生成数。
- 订单状态统计。
- 热门产品点击排行。
- AI 调用失败次数。

### 5.3 热卖产品管理

字段包括：

- 开发周期。
- 推荐描述。

包含分类管理和文章管理。

文章字段包括：标题、分类、摘要、正文、标签、排序、状态。

### 5.5 教程管理

### 5.6 销售话术库

话术类型：

- 催单话术

- 成交话术

支持按类型检索和启停。

### 5.7 订单管理

订单状态：

- 待沟通
- 已报价
- 待付款
- 开发中
- 已完成

订单可由后台手动创建，也可由报价结果生成草稿。

### 5.8 AI 配置

支持配置：

- DeepSeek API Key。
- 模型名。
- 温度参数。
- 超时时间。
- 风控系统提示词。
- 违禁词列表。
- 兜底话术。

后台编辑 API Key 时默认掩码展示，后端日志不打印完整 Key。

## 6. 后端设计

后端使用 Node.js + Express，连接 MySQL。

推荐结构：

```text
backend/
├─ src/
│  ├─ app.js
│  ├─ config/
│  ├─ db/
│  ├─ middleware/
│  ├─ routes/
│  ├─ controllers/
│  ├─ services/
│  └─ utils/
└─ package.json
```

核心服务：

- `aiService`：封装 DeepSeek 调用、超时、失败兜底。
- `riskService`：系统提示词拼接、输入输出违禁词过滤。
- `quoteService`：识别需求，生成报价、周期、功能清单。
- `planService`：生成建站方案、页面结构、功能规划。
- `searchService`：统一搜索产品、文章、教程、历史。
- `toolService`：工具箱统一入口，支持真实工具和预留工具。
- `statsService`：统计今日使用次数、回复生成数、订单状态。

## 7. 核心数据流

### 7.1 AI 智能回复

```text
小程序输入客户问题
→ POST /api/ai/chat
→ riskService 拼接平台风控系统提示词
→ aiService 调用 DeepSeek
→ riskService 二次过滤 AI 输出
→ 保存 AI 使用记录
→ 返回安全客服回复
```

### 7.2 智能报价

```text
用户填写需求
→ POST /api/quote/generate
→ quoteService 识别需求类型
→ 匹配产品和报价规则
→ DeepSeek 润色销售表达
→ 返回报价范围、开发周期、功能清单
→ 可生成订单草稿
```

### 7.3 建站方案生成

```text
用户输入行业、预算、功能需求
→ POST /api/plan/generate
→ planService 生成页面结构、功能清单、开发周期
→ DeepSeek 润色为客户可读方案
→ 保存历史记录
→ 返回方案
```

## 9. 统一响应与错误处理

成功响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

失败响应：

```json
{
  "code": 40001,
  "message": "参数错误",
  "data": null
}
```

错误码：

- `40001` 参数错误。
- `40101` 后台登录失效。
- `40301` 无权限。
- `40401` 数据不存在。
- `42901` 请求过于频繁。
- `50001` 服务内部错误。
- `50002` AI 服务异常。
- `50003` AI 响应超时。
- `50004` 内容触发风控。

前端处理：

- 普通接口失败显示 Toast 或消息提示。
- AI 调用失败返回可复制的兜底回复。
- 搜索无结果显示空状态和推荐入口。
- 未接入工具显示“功能已预留，即将接入”。
- 后台保存失败时保留表单输入内容。
- 后台删除操作必须二次确认。

## 10. AI 风控机制

### 10.1 系统提示词

后端调用 DeepSeek 前强制拼接系统提示词：

```text
你是 AI小福，一个用于电商平台建站接单场景的客服助手。
回复必须适合在闲鱼、淘宝、拼多多等平台内沟通。
不得引导客户添加微信、QQ、手机号、邮箱或跳转站外链接。
不得承诺违规服务，不得诱导绕过平台规则。
表达要专业、简短、自然，优先促成平台内继续沟通。
```

### 10.2 违禁词过滤

输入和输出均检查违禁词。默认覆盖：

- 微信、wx、VX、QQ、手机号。
- 加我、私聊、站外、线下交易。
- 绕过平台、规避监管。
- 后台配置的其他敏感词。

触发策略：

- 输入触发：替换敏感内容后继续请求，或提示用户调整输入。
- 输出触发：使用安全兜底话术替代。

默认兜底话术：

```text
这个需求可以的，我先根据你描述的功能帮你整理一版建站方案和报价范围，你可以继续在平台里补充行业、页面数量和功能要求，我会帮你细化。
```

### 10.3 超时与失败兜底

- DeepSeek API 超时时间为 5 秒。
- 超时、报错、Key 缺失时返回安全兜底回复。
- AI 调用失败原因写入 `ai_usage_logs`。
- 后台仪表盘展示 AI 失败次数。

## 13. 实施边界

第一阶段只保证核心 MVP 可运行和可演示。Whois、备案查询、测速、SEO 检测等工具会保留统一入口和接口结构，但真实第三方服务接入放到后续迭代。这样可以先完成 AI 回复、报价、方案、知识库、产品、订单这些直接支撑建站接单的核心能力。
